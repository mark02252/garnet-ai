## Research Memory 브레인스토밍 진행 내용

### 확정된 방향

**Q1. 어떤 종류의 리서치?**
→ **C. 둘 다** — 외부 정보(아티클/리포트/트렌드) + 내부 인사이트(캠페인 발견/실험 결과)
→ 이유: AGI 실현을 위해서는 외부 지식과 내부 경험 모두 필요

**Q2. 어떻게 활용?**
→ 아직 미결 (다음 질문으로 진행 중)
→ 제안된 3가지 접근법:

| 방식 | 설명 | 평가 |
|---|---|---|
| A. 지식 베이스형 | 노션 같은 저장/검색, AI는 요약만 | AGI 방향과 거리 있음 |
| B. AI 컨텍스트 주입형 | 벡터 DB + 임베딩, 자동 주입 | 복잡, 오버엔지니어링 위험 |
| C. 태그 기반 자동 검색형 | 태그+키워드 매칭으로 AI 컨텍스트 주입 | **추천** |

### 추천 방향: C (태그 기반 자동 검색형)
- 저장: 제목/내용/태그/출처/타입(외부/내부)
- 검색: 태그+키워드 매칭 → AI 컨텍스트 주입
- 벡터 DB 없이 구현 가능
- 나중에 벡터 검색 확장 가능
- 기존 `/learning` 페이지와 역할 분리 필요

### 미결 질문들 (재개 시 순서대로)
1. C 방향 확정 여부 확인
2. `/learning`과의 차이점/역할 분리 명확화
3. AI 주입 시점 (브리프 작성? 캠페인 실행? 모두?)
4. 저장 방식 (수동 입력? URL 저장? 자동 수집?)
5. 데이터 모델 설계
