# Post 3: 구매 여정 이탈 분석

## 본문

구매 여정에서 53%가 이탈하는 지점을 찾았습니다.

영화관 예매 프로세스를 6단계로 쪼개서 추적해봤습니다.

① 영화 목록 조회
② 예매 페이지 진입
③ 상영시간 선택
④ 좌석 확정
⑤ 결제수단 선택 ← 여기서 53% 이탈
⑥ 결제 완료

"결제수단 선택"에서 절반 이상이 나갑니다.

문제는, 이걸 알기까지가 더 어려웠다는 겁니다.

기존에는 GA4에서 "결제 시작"과 "결제 완료"만 추적하고 있었습니다.
중간 단계가 없으니 "어디서 빠지는지" 알 수 없었습니다.

해결:
• GTM 이커머스 이벤트를 6단계로 재설계
• 기존 페이지뷰 기반 트리거를 실제 사용자 액션 기반으로 전환
• GTM API로 태그 11개를 일괄 수정 (수동 작업 0)

결과:
• 이탈 지점이 "결제수단 선택" 단계임을 정확히 특정
• 에이전트가 매일 아침 퍼널 이탈률을 자동 분석해서 보고

"데이터를 모으는 것"과 "데이터가 말하게 하는 것"은 다른 문제입니다.
