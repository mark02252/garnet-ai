# Post 4: GTM 자동화

## 본문

GTM 태그 11개를 코드 한 줄로 고쳤습니다.

Google Tag Manager를 마케터라면 대부분 웹 UI에서 클릭으로 관리합니다.
저도 그랬는데, 문제가 생겼습니다.

GTM 감사를 돌려보니:
• 이커머스 태그 3개가 "페이지뷰" 트리거로 잘못 설정
• 결제 이벤트 2개가 아예 없음
• 매출 금액(value)이 0으로 전달되는 버그
총 11건의 문제.

웹 UI에서 하나씩 고치면 30분.
실수 가능성도 있고, 기록도 안 남습니다.

그래서 GTM API를 연결하고 스크립트를 만들었습니다.

1. 감사 스크립트: 모든 태그/트리거/변수를 스캔해서 문제점 자동 리포트
2. 수정 스크립트: Dry-run으로 미리보기 → 확인 후 일괄 적용
3. 게시 스크립트: 코드로 바로 Publish

결과:
• 11건 문제 → 3분 만에 전부 수정 + 프로덕션 반영
• 모든 변경이 코드로 기록되어 추적 가능
• 다음에 또 문제 생기면 감사 스크립트만 돌리면 됨

마케터가 코드를 잘 몰라도,
"내가 관리하는 태그가 정상인지" 확인하는 체계는 필요합니다.

수동으로 30분 vs 자동으로 3분.
이 차이가 쌓이면 시스템이 됩니다.
