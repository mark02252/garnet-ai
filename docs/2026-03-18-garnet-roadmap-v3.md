## Phase 1: 시각화 + 실제 연동 (v0.3.0)

목표: **"만들고 → 예약하고 → 실제로 올라가고 → 결과가 그래프로 보인다"**

### 1-1. 대시보드 재설계 + 도달 추이 그래프

**문제:** 현재 `/dashboard`는 플레이북 통계만 표시. 마케팅 성과를 한눈에 볼 수 없음.

**목표:**
- KPI 달성률 카드 (목표 대비 현재값, 진행률 바)
- Instagram 도달 추이 30일 라인 차트 (recharts)
- 캠페인 실행 건수 + SNS 게시 건수 카드
- 최근 게시물 Top 5 (도달 기준)

**변경 파일:**
- `app/dashboard/page.tsx` — 전면 재설계
- 새 컴포넌트: 차트 위젯

**기술:** recharts (이미 사용 가능하거나 추가), Prisma 쿼리 집계

**예상 난이도:** 낮음

---

### 1-2. Instagram 실제 게시 구현

**문제:** `POST /api/sns/schedule/process`가 DB 상태만 변경하고 실제 Instagram Graph API 발행을 하지 않음.

**목표:**
- 이미지 게시: Graph API `/{ig-user-id}/media` → `/{ig-user-id}/media_publish`
- 캐러셀 게시: 슬라이드별 `/{ig-user-id}/media` (is_carousel_item) → container → publish
- 텍스트만 게시: 캡션 포함 이미지 필수 (Instagram 정책)
- 게시 성공/실패 상태 반영 + 에러 메시지 저장

**변경 파일:**
- `lib/sns/instagram-publisher.ts` — 새 파일, Graph API 발행 로직
- `app/api/sns/schedule/process/route.ts` — 실제 발행 호출 연동
- `app/api/sns/content/[id]/publish/route.ts` — 즉시 발행 엔드포인트 (신규)

**필요 권한:** `instagram_content_publish` (이미 스코프에 포함)

**예상 난이도:** 중간

---

### 1-3. 자동 스케줄 실행 (크론)

**문제:** 예약 게시를 수동으로 process 엔드포인트를 호출해야 함. 자동 실행 없음.

**목표:**
- Electron 백그라운드 타이머: 1분마다 `schedule/process` + `schedule/missed` 호출
- 실행 로그 (성공/실패/스킵) 저장
- 운영 허브 타임라인에 게시 이벤트 표시

**변경 파일:**
- `electron/main.ts` — 백그라운드 스케줄러 추가
- `app/api/sns/schedule/process/route.ts` — 1-2에서 완성된 실제 발행 사용

**예상 난이도:** 낮음

## Phase 2: AI 자동화 (v0.3.1)

목표: **"AI가 성과를 분석하고, 개선 방향을 제시하고, 콘텐츠까지 만들어준다"**

### 2-1. AI 성과 분석 리포트

**문제:** 분석 데이터는 있으나 인사이트가 숫자 나열 수준. 왜 성과가 좋은지/나쁜지, 무엇을 해야 하는지 알 수 없음.

**목표:**
분석 실행 시 AI가 자동으로 생성하는 종합 리포트:

1. **성과 요약** — 전체 도달/참여율 추이, 전주 대비 변화
2. **Top 게시물 분석** — 상위 5개 게시물 + 왜 잘 됐는지 (시간대, 유형, 해시태그, 캡션 패턴)
3. **저성과 진단** — 하위 5개 + 개선 포인트
4. **패턴 인사이트** — 최적 게시 시간대, 콘텐츠 유형별 평균 성과, 키워드/해시태그 효과
5. **추천 게시물** — 다음에 올리면 좋을 콘텐츠 주제 3~5개 (근거 포함)
6. **광고 예산 제안** — 부스트 추천 게시물 + 예상 예산 + 기대 효과

**데이터 소스:**
- `SnsAnalyticsSnapshot` — 일별 도달/참여
- `fetchInstagramMediaInsights()` — 게시물별 상세 인사이트 (이미 구현)
- `SnsScheduledPost` — 게시 이력
- `SnsContentDraft` — 콘텐츠 유형/주제

**변경 파일:**
- `lib/sns/performance-analyzer.ts` — 데이터 수집 + LLM 분석 파이프라인
- `app/api/sns/analytics/report/route.ts` — 리포트 생성 엔드포인트
- `app/sns/analytics/page.tsx` — 리포트 뷰 UI (기존 페이지 확장)
- Prisma 모델: `SnsPerformanceReport` — 리포트 저장

**예상 난이도:** 중간

---

### 2-2. 추천 콘텐츠 → 스튜디오 초안 연동

**문제:** 리포트에서 "이런 콘텐츠를 올리세요"라고 추천만 하면 사용자가 직접 스튜디오로 가서 다시 만들어야 함.

**목표:**
- 리포트 내 추천 항목마다 **[이 추천으로 콘텐츠 만들기]** 버튼
- 클릭 시 추천 주제 + 페르소나 컨텍스트로 `POST /api/sns/content` 호출
- 자동 생성된 초안이 스튜디오에 DRAFT 상태로 저장
- 사용자는 편집 → 예약 → 자동 게시 플로우로 이어감

**변경 파일:**
- `app/sns/analytics/page.tsx` — 추천 항목에 버튼 추가
- `app/api/sns/content/route.ts` — 추천 기반 생성 모드 지원

**예상 난이도:** 낮음 (2-1 완료 후)

## Phase 3: 광고 + 콘텐츠 품질 (v0.3.2)

목표: **"광고까지 자동화하고, 콘텐츠 퀄리티를 올린다"**

### 3-1. Meta 광고 API 연동

**문제:** 광고 예산 제안은 하지만 실제 집행은 수동. Meta Ads Manager에서 직접 해야 함.

**목표:**

**기본 기능:**
- 광고 캠페인 생성 (목표: 도달/트래픽/참여)
- 기존 게시물 부스트 (가장 흔한 사용 사례)
- 타겟 설정 (지역, 연령, 관심사)
- 예산 + 기간 설정
- 광고 성과 조회 (도달, 클릭, 비용, ROAS)

**UI:**
- `/sns/ads` 신규 페이지 — 광고 관리 대시보드
- 게시물 상세에서 "이 게시물 부스트" 버튼
- AI 리포트에서 "추천 예산으로 광고 생성" 연동

**필요 API:**
- Marketing API: `/{ad-account-id}/campaigns`, `/adsets`, `/ads`
- 필요 권한: `ads_management`, `ads_read` (별도 앱 심사 필요할 수 있음)

**변경 파일:**
- `lib/sns/meta-ads-api.ts` — 광고 API 래퍼
- `app/sns/ads/page.tsx` — 광고 관리 UI
- `app/api/sns/ads/` — CRUD 엔드포인트
- Prisma 모델: `SnsAdCampaign`, `SnsAdSet`, `SnsAd`, `SnsAdInsight`

**예상 난이도:** 높음

- `lib/sns/video-renderer.ts` — 실제 구현

- `app/sns/studio/[draftId]/page.tsx` — 영상 미리보기 + 내보내기 버튼

## 비전

Garnet은 **1인 마케터 또는 소규모 팀이 AI와 함께 마케팅 전 과정을 운영하는 데스크톱 OS**다.

- **전략:** 캠페인 스튜디오 + 세미나 시뮬레이션으로 의사결정
- **실행:** SNS 스튜디오에서 AI 콘텐츠 생성 → 자동 게시 → 광고 집행
- **분석:** Instagram 인사이트 + AI 리포트 → 다음 액션 추천 → 콘텐츠 자동 생성
- **학습:** 성과 데이터가 플레이북으로 축적 → 점점 더 정확한 추천

**최종 목표:** "브리핑 한 줄 입력 → AI가 콘텐츠 제작 → 최적 시간에 자동 게시 → 성과 분석 → 개선 콘텐츠 재생성" 전 과정 자동화.
